Перед выполнением заданий важно самостоятельно повторить шаги, выполняемые на уроке.

1) Создать текстовый файл.
touch text.txt

2) Установить владельцем и группой владельцев суперпользователя root.
sudo chmown root:root text.txt

3) Запретить всем остальным доступ к файлу и проверить что доступ пропал.
sudo chmod 770 text.txt
sudo chmod o-rwx text.txt

4) Установить маску создания файлов так, чтобы у владельца были полные права, а у остальных никакого.
Проверить работу маски.
umask 077
Проверить работу маски.
touch testfile
ls -l testfile  # Ensure the permissions are -rwx------

### Маска создания файлов (umask)
**Маска создания файлов (umask)** — это настройка, которая определяет права доступа по умолчанию для новых файлов и директорий, создаваемых пользователем.
Она задает, какие права будут отключены при создании нового файла или директории.
- **umask 077**: Устанавливает маску, при которой у владельца будут полные права (rwx), а у всех остальных не будет никаких прав.
- Пример: Если umask установлена в 077, то новый файл будет создан с правами `-rw-------`, а новая директория с правами `drwx------`.


5) Создать папку и сделать владельцем суперпользователя.
mkdir folder
sudo chmown root:root folder

6) Разрешить всем полный доступ к папке.
sudo chmod 777 folder
sudo chmod ugo+rwx folder

7) Установить на папку sticky-бит, убедиться что никто кроме владельца не может изменить ее структуру.
sudo chmod +t folder
ls -ld folder  # Ensure the permissions include 't' (drwxrwxrwt)

### Sticky-бит
**Sticky-бит** — это специальный бит, который устанавливается на директорию и ограничивает удаление или переименование файлов в этой директории только их владельцами или суперпользователем (root).
- **sudo chmod +t folder**: Устанавливает sticky-бит на директорию `folder`.
- Пример: Если sticky-бит установлен на директорию, то даже если у всех пользователей есть права на запись в эту директорию, они смогут удалять или переименовывать только свои собственные файлы.
Sticky-бит часто используется в общих директориях, таких как `/tmp`, чтобы предотвратить удаление файлов другими пользователями.
